# Rider Imagination System

## Goal

Riders are toy bikes brought to life by imagination.

They should not be fixed real rider personalities. They should feel like fresh imaginary racers every time the player presses Play Race.

## Random Rider Generation

Each race should generate toy riders with random:

- Name or color identity.
- Number.
- Bike color.
- Temporary skill profile.
- Aggression.
- Consistency.
- Jump skill.
- Whoop skill.
- Sand skill.
- Roller skill.
- Hill skill.
- Start skill.

Names and skills must not be permanently tied together. A rider named Ace should not always be brave, and a blue bike should not always be fast.

## Rider Behavior

Races should create little stories naturally:

- Some riders get good starts.
- Some riders roll big jumps.
- Some riders attempt risky jumps and crash.
- Some riders clear triples.
- Some riders struggle in whoops or sand.
- Some riders become the imaginary hero of that one race.

## Skill Interpretation

`start_skill` affects holeshot variation.

`jump_skill` affects singles, doubles, triples, and tabletops.

`whoop_skill` affects rhythm through whoops.

`sand_skill` affects soft sections.

`roller_skill` affects smooth rolling sections.

`hill_skill` affects climbs and larger elevation features.

`aggression` affects whether a rider attempts risky features.

`consistency` affects wobble, mistakes, and speed stability.

## Toy-Like Motion

Rider motion should feel playful and readable:

- Smooth lane changes.
- Stable lane offsets.
- No constant jitter.
- Gentle crashes or spinouts.
- Small hop arcs.
- Airborne riders draw above obstacles.
- Grounded riders stay close to the track.

This should not become full real-life motorcycle physics.
