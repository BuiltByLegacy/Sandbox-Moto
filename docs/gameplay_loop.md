# Gameplay Loop

Build track -> run toy race -> get imagination feedback -> modify track -> race again.

## 1. Build Track

The player draws and shapes a smooth motocross track in warm sand. Building is the main mechanic and should feel relaxed, tactile, and playful.

The player can:

- Draw a smooth track path.
- Place a start gate.
- Place a finish.
- Add jumps and rhythm sections.
- Add sand, rollers, whoops, hills, and toy-like features.
- Use a dozer or shovel-style tool to adjust the imagined track.

## 2. Run Toy Race

The player presses Play Race. Tiny toy riders spawn with random names, numbers, colors, and skill profiles.

The race is not a full simulation. It is a playful validation pass that shows how the imagined track rides.

## 3. Get Imagination Feedback

After the race, the game gives playful feedback bubbles:

- Blue bike almost cleared the triple!
- Red bike loved the whoops.
- Green bike got stuck in the sand.
- That double might be too big.

Feedback should sound like a kid narrating an imaginary race, not like a data report.

## 4. Modify Track

The player returns to build mode. They can reshape the track, move obstacles, add features, or simplify a section.

There is no failure state. The race simply gives the player ideas.

## 5. Race Again

The player runs another race and sees a new story. Names and skills should randomize each race so the same track can produce fresh toy-moto moments.
