# Building Tools

## Goal

Tools should feel like playing in dirt, not engineering.

The player should feel like they are shaping sand with hands, toys, and simple objects.

Play Time should feel like dragging fingers through sand: loose, tactile, reversible, and inviting.

## Core Tools

- Shovel.
- Rake.
- Hand smoothing.
- Bucket.
- Water.
- Toy bulldozer.
- Berm tool.
- Jump mold.
- Whoop stamp.
- Roller stamp.
- Start gate.
- Finish marker.
- Flags and cones.
- Dozer or flatten tool.

## Track Features

- Singles.
- Doubles.
- Triples.
- Tabletops.
- Whoops.
- Rollers.
- Sand.
- Hills.
- Berms.
- Ruts.
- Puddles later.
- Tire tracks later.

## Tool Feel

Tools should be direct and playful:

- Drag to smooth.
- Stamp to place.
- Pull to raise.
- Push to flatten.
- Drop flags and cones like toys.
- Pile dirt.
- Smooth dirt.
- Flatten.
- Carve ruts.
- Shape berms.
- Water sections.
- Repair jumps.
- Knock down and rebuild.

Avoid making tools feel like CAD, a professional terrain editor, or a business management interface.

## Track Editing Baseline

Keep the current free-draw smooth track system as the baseline for now.

Do not switch back to a blocky grid system. If grid logic is introduced later, it should only guide editing while visuals remain smooth and natural.
